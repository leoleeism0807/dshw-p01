## 存储方式

- **基础**：CSV（方式 A）——所有原始数据和清洗后数据均以 CSV 存储
- **进阶**：Parquet（方式 B）——清洗后数据额外保存 Parquet 格式

**选择 Parquet 的理由**：
1. Parquet 是列式存储格式，读取特定列时效率远高于 CSV
2. 自带 Schema 信息，避免 CSV 读取时的类型推断问题
3. 压缩效率高，文件体积更小
4. 是大数据生态（Spark、Dask）的标准格式，学习价值高

**CSV 格式的优缺点**：
- **优点**：人类可读、通用性强、几乎所有工具都支持、便于调试
- **力不从心的场景**：数据量大时读取慢、无类型信息（日期/数值需手动转换）、文件体积大、不支持列式读取
